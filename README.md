# GitHub Release Notifier

Професійний мікросервіс для відстеження релізів у GitHub репозиторіях з автоматичним email-сповіщенням. Проєкт реалізовано як монолітний додаток мовою Go з підтримкою гібридної архітектури.

## Особливості (Features)
* **Гібридний сервер:** Одночасна обробка REST та gRPC запитів на одному порту (8081) через мультиплексор cmux.
* **API на Go (Chi):** Швидкий та надійний HTTP-сервер.
* **PostgreSQL:** Повноцінна база даних для зберігання підписок з підтримкою транзакцій.
* **Автоматичні міграції:** База налаштовується сама при старті (golang-migrate).
* **Фоновий сканер:** Автономно перевіряє GitHub API кожні 5 хвилин (Goroutines).
* **Redis кешування:** Оптимізація перевірки існування репозиторіїв з TTL 10 хвилин.
* **Обробка Rate Limits:** Інтеграція GitHub Token (до 5000 запитів/год) та коректна обробка 403/429 статусів.
* **Security:** Доступ до API захищений через API-Key (Custom Middleware).
* **Метрики (Prometheus):** Додано ендпоїнт /metrics для телеметрії та підрахунку викликів зовнішніх API.
* **Docker & Compose:** Запуск всього проєкту (App, DB, Redis) однією командою.
* **CI/CD:** Налаштовані GitHub Actions для автоматичного тестування.
* **Documentation:** Додано Swagger (OpenAPI) опис у папці docs.
* **Деплой:** Сервіс налаштовано для роботи в хмарі (Render) з готовою статичною UI-сторінкою.

## Змінні оточення (Environment Variables)
Для запуску проекту необхідно вказати наступні змінні:
* `DB_URL` - Рядок підключення до PostgreSQL
* `REDIS_URL` - Адреса підключення до Redis (за замовчуванням localhost:6379)
* `GITHUB_TOKEN` - Personal Access Token (PAT) GitHub
* `API_KEY` - Секретний ключ для доступу до API
* `SMTP_USER` - Дані для розсилки (Email)
* `SMTP_PASS` - Дані для розсилки (App Password)

## Швидкий старт (Локально)
1. Переконайтеся, що у вас встановлено Docker Desktop.
2. Склонуйте проєкт та виконайте в терміналі:
   ```bash
   docker-compose up -d --build
   ```
3. Сервер запуститься на порту 8081.

## Авторизація та Використання
Для виконання запитів до `POST /api/subscribe` необхідно передавати заголовок авторизації.

Приклад cURL запиту:
```bash
curl -X POST http://localhost:8081/api/subscribe \
  -H "Content-Type: application/json" \
  -H "X-API-Key: my-secret-key" \
  -d '{"email": "user@example.com", "repository": "golang/go"}'
```
